System Ambiguities Ambigüedades del Sistema

List of questions and points to clarify before starting development. Listado de dudas y puntos a aclarar antes de iniciar el desarrollo.

Context Contexto

Before starting development, it is essential that the entire team shares a unified vision of the system's rules. This page gathers the identified questions to discuss them and define the acceptance criteria for each story. Antes de comenzar el desarrollo, es fundamental que todo el equipo comparta una misma visión sobre las reglas del sistema. Esta página recopila las dudas identificadas para discutirlas y definir los criterios de aceptación de cada historia.

Questions Preguntas

Schedule and Appointment Management Gestión de Agenda y Citas

  • Can a doctor have multiple non-contiguous work periods (e.g., Jan-Mar and then Jun-Aug)?¿Un médico puede tener múltiples períodos de trabajo no contiguos (ej. enero-marzo y junio-agosto)?
  • If a doctor's work period is modified or deleted, what happens to already scheduled appointments?Si se modifica o elimina el período de trabajo de un doctor, ¿qué sucede con las citas ya programadas?
  • Is the "timetable" a reusable template, or is a new one always created for each work period?¿El "horario" es una plantilla reutilizable o siempre se crea uno nuevo para cada período de trabajo?
  • How are conflicts handled if an attempt is made to create an overlapping period?¿Cómo se gestionan los conflictos si se intenta crear un período que se solapa con otro existente?
  • Can appointments be scheduled on holidays if they fall within the doctor's work period?¿Se pueden crear citas en días festivos si caen dentro del período laboral del doctor?

Appointment Details and Notifications Detalles de la Cita y Notificaciones

  • What is the default duration of an appointment? Is it configurable?¿Qué duración por defecto tiene una cita? ¿Es configurable?
  • Is email the only notification mechanism for urgent appointments? What exact information does it contain?¿La notificación de urgencia por e-mail es el único mecanismo? ¿Qué información exacta contiene?
  • What happens if sending the urgent email fails? Are there retries?¿Qué ocurre si falla el envío del correo electrónico de urgencia? ¿Hay reintentos?
  • Besides the doctor's availability, is the office (location) availability also checked?Además de la disponibilidad del doctor, ¿se comprueba la disponibilidad de la consulta (ubicación)?

Data Model and Relationships Modelo de Datos y Relaciones

  • (1:N) Doctor <> Work Period:(1:N) Médico <> Período Trabajo: Can a doctor have multiple simultaneous work periods? ¿Un médico puede tener varios períodos de trabajo asignados simultáneamente?
  • (1:N) Appointment <> Patient/Doctor:(1:N) Cita <> Paciente/Doctor: Can an appointment exist without an assigned patient (e.g., a "reserved slot")? Cardinality (0..1) to Patient. ¿Puede existir una cita sin paciente asignado (ej. un "hueco reservado")? Cardinalidad (0..1) a Paciente.
  • (1:N) Patient <> Appointment Contact:(1:N) Paciente <> Contacto Cita: Is the contact information an attribute of the appointment or a separate entity? If it's an entity, can a patient have a list of associated contacts? ¿La información de contacto es un atributo de la cita o una entidad separada? Si es una entidad, ¿un paciente puede tener una lista de contactos asociados?
  • (N:M) Appointment <> Diagnosis:(N:M) Cita <> Diagnóstico: Can a diagnosis (e.g., "flu") be associated with multiple appointments, or is each diagnosis a unique record linked to a single appointment? ¿Un diagnóstico (ej. "gripe") puede asociarse a múltiples citas, o cada diagnóstico es un registro único ligado a una sola cita?
  • (N:M) Appointment <> Prescription:(N:M) Cita <> Prescripción: Does a prescription belong exclusively to one appointment, or can it be reused or referenced from others? ¿Una prescripción pertenece exclusivamente a una cita o puede reutilizarse o referenciarse desde otras?
  • (1:N) User <> Role:(1:N) Usuario <> Rol: Are roles a separate entity or a simple attribute in the user table? Can a user have multiple roles? ¿Los roles (doctor, manager) son una entidad propia o un simple atributo en la tabla de usuarios? ¿Un usuario puede tener múltiples roles?
  • Leave Management:Gestión de Ausencias: Is leave (e.g., vacation) managed by creating a "non-work period" or is it a separate entity that blocks the schedule? ¿Las vacaciones se gestionan creando un "período de no trabajo" o es una entidad aparte que bloquea la agenda?
