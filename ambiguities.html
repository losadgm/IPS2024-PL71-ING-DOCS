Discusión de Ambigüedades del Proyecto

Preguntas clave para definir el Modelo de Datos y la Lógica de Negocio

Contexto

Antes de comenzar el desarrollo, es fundamental que todo el equipo comparta una misma visión sobre las reglas del sistema. Esta página recopila las dudas identificadas para discutirlas y definir los criterios de aceptación de cada historia.

Preguntas

Gestión de Agenda y Citas

  • ¿Un médico puede tener múltiples períodos de trabajo no contiguos (ej. enero-marzo y junio-agosto)?
  • Si se modifica o elimina el período de trabajo de un doctor, ¿qué sucede con las citas ya programadas?
  • ¿El "horario" es una plantilla reutilizable o siempre se crea uno nuevo para cada período de trabajo?
  • ¿Cómo se gestionan los conflictos si se intenta crear un período que se solapa con otro existente?
  • ¿Se pueden crear citas en días festivos si caen dentro del período laboral del doctor?

Detalles de la Cita y Notificaciones

  • ¿Qué duración por defecto tiene una cita? ¿Es configurable?
  • ¿La notificación de urgencia por e-mail es el único mecanismo? ¿Qué información exacta contiene?
  • ¿Qué ocurre si falla el envío del correo electrónico de urgencia? ¿Hay reintentos?
  • Además de la disponibilidad del doctor, ¿se comprueba la disponibilidad de la consulta (ubicación)?

Modelo de Datos y Relaciones

  • (1:N) Médico <> Período Trabajo: ¿Un médico puede tener varios períodos de trabajo asignados simultáneamente?
  • (1:N) Cita <> Paciente/Doctor: ¿Puede existir una cita sin paciente asignado (ej. un "hueco reservado")? Cardinalidad (0..1) a Paciente.
  • (1:N) Paciente <> Contacto Cita: ¿La información de contacto es un atributo de la cita o una entidad separada? Si es una entidad, ¿un paciente puede tener una lista de contactos asociados?
  • (N:M) Cita <> Diagnóstico: ¿Un diagnóstico (ej. "gripe") puede asociarse a múltiples citas, o cada diagnóstico es un registro único ligado a una sola cita?
  • (N:M) Cita <> Prescripción: ¿Una prescripción pertenece exclusivamente a una cita o puede reutilizarse o referenciarse desde otras?
  • (1:N) Usuario <> Rol: ¿Los roles (doctor, manager) son una entidad propia o un simple atributo en la tabla de usuarios? ¿Un usuario puede tener múltiples roles?
  • Gestión de Ausencias: ¿Las vacaciones se gestionan creando un "período de no trabajo" o es una entidad aparte que bloquea la agenda?
